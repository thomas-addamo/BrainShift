# Canovaccio `docs/`

Questa cartella contiene **i file da copiare nel vostro repository** come scheletro della documentazione.

## Cosa fare

1. Copiate **tutti** i file `.md` di questa cartella nella cartella `docs/` del vostro repository GitHub (non la sottocartella `docs-template/`: direttamente `docs/`).
2. **Non consegnate questo file `README.md`**: serve solo a voi come istruzioni, non va nel repository.
3. Compilate ciascun file man mano che il progetto procede. Non lasciate tutta la documentazione all'ultimo giorno.
4. **Rimuovete** le sezioni «Domande-guida» dai file finali: servono solo come traccia di riflessione.
5. Se un file non si applica al vostro progetto (es. `personalizzazioni.md` se non avete personalizzato), cancellatelo.

## File del canovaccio

- `README-progetto.md` — overview pubblico del progetto. Rinominatelo in `docs/README.md` se preferite, o lasciate `README-progetto.md` e tenete un README minimale alla radice del repo con i link alle pagine di `docs/`.
- `architettura.md` — struttura interna del progetto
- `scelte.md` — scelte implementative, decisioni, alternative scartate
- `uso-ia.md` — dichiarazione granulare sull'uso degli assistenti IA
- `devlog.md` — diario di bordo settimanale
- `personalizzazioni.md` — solo se avete personalizzato il gioco

## Cosa può fare l'IA e cosa no

L'IA può aiutarvi con il **codice**. Per la documentazione in `docs/`:

- può correggervi refusi e grammatica
- **non** può scrivere al posto vostro `scelte.md`, `devlog.md`, `uso-ia.md`

Queste sono le parti metacognitive: se le scrive l'IA, perdete tutta la parte formativa del progetto. E all'orale si nota.
